Recursion

we have all seen various posts about recession on how it makes DRY code, makes it more readable and all around an awesome thing, but how does it work ? Basically you are calling the method/function in itself. The easiest way to explain recursion is using the factorial problem.

PSEUDOCODE 

DEFINE METHOD recursion(1 PARAMETER: n (integer value))
    IF n less than or equal to 0; n<=0;
        return 1
    ELSE
        n * recursion(n-1)
    END IF-ELSE LOOP

So if you give out an value of 6 to the recursion method, this is what will happen

    IF n(6) less than or equal to 0; n<=0; #no goto else
        return 1
    ELSE
